##B
###Best Time
A feature that recommends the most optimal time to send a message to each user for a batch campaign or a journey. For more information, refer to [Best Time](https://docs.clevertap.com/docs/best-time).

###Billing 
Billing covers your plan, billing details, card details, and invoices. For more information on our different plans and billing, refer to our [pricing page](https://clevertap.com/pricing/).

###Boards
A [collection of cards](https://docs.clevertap.com/docs/simple-reports) containing key metrics.

###Bulletins
[Automated messages](https://docs.clevertap.com/docs/bulletins) that are created for a specific segment for recurring business events in CleverTap.

##C
###Campaigns
A collection of tactics designed to capture the attention of its targets through different messaging channels. For more information, refer to [Campaign Optimization](https://docs.clevertap.com/docs/intro-to-campaigns).

###Campaign Limits
[Limits](https://docs.clevertap.com/docs/push#push-campaign-throttling) to determine how many messages a user receives per day.

###Channels 
The type of communication to reach your users, including:
  * [Push Notifications](https://docs.clevertap.com/docs/push)
  * [In-App](doc:in-app)
  * [App Inbox](https://docs.clevertap.com/docs/app-inbox)
  * [Native Display](https://docs.clevertap.com/docs/native-display)
  * [Web Push](https://docs.clevertap.com/docs/web-push)
  * [Web Pop-up](https://docs.clevertap.com/docs/web-pop-ups)
  * [SMS](doc:sms-overview)
  * [Email](https://docs.clevertap.com/docs/email)
  * [Google Adwords](https://docs.clevertap.com/docs/google-adwords)
  * [Facebook Audiences](https://docs.clevertap.com/docs/facebook-audiences)
  * [WhatsApp](https://docs.clevertap.com/docs/whatsapp)
  * [Webhooks](https://developer.clevertap.com/docs/webhooks)

###Compliance & Certifications
A list of all [compliances and certifications](https://docs.clevertap.com/docs/compliance-certifications) applicable to CleverTap.

###Composite Events
Combined multiple events together. For more information, refer to [Composite Events](https://docs.clevertap.com/docs/composite-events).

###Control Groups
In a marketing A/B testing, members of a control group receive a standard treatment. Control groups are a set of users marked to be excluded from all marketing campaigns, so the primary purpose is that marketers can measure the efficacy of a campaign. 

Note that you can have more than one treatment group, more than one control group, or both. For more information, refer to [Control Groups](https://docs.clevertap.com/docs/control-groups).

###Conversations
A way to communicate between brands and users. For more information, refer to [Conversations](https://docs.clevertap.com/docs/conversations).

##E
###Events
Specific, individual actions that users perform in your app or website over a period of time as they progress down the conversion funnel. For more information, refer to [Events](https://docs.clevertap.com/docs/events).

##J
###Journeys
A visual campaign builder that lets you create omnichannel messaging experiences for your users. For more information, refer to [Journeys](https://docs.clevertap.com/docs/journeys).

##L
###Lifecycle Optimizer
A [tool](https://docs.clevertap.com/docs/lifecycle-optimizer) that enables marketers to influence users into progressive lifecycle stages. 

###Linked Content
The ability to pull content via an API from a web server, third-party tool, or in-house software for personalization. For more information, refer to [Linked Content](https://docs.clevertap.com/docs/linked-content).

##M
###Monthly Active Users (MAUs)
The total number of app users in a month.

##O
###Organization
The organization section provides information on your projects, billing, single sign-on, and CleverTap compliances and certifications.

##P
###Product Experiences
A product configuration (also known as an A/B test) to modify app behavior and feature flags to add or remove features from your app without performing an app store deployment. For more information, refer to [Product Experiences](https://docs.clevertap.com/docs/product-experiences).

###Project
A shared workspace for teams to track and manage engagement for a mobile app or website. For more information, refer to [Project Setup](https://docs.clevertap.com/docs/project-setup).

### Pull Notification
A feature that enhances the deliverability of push notifications to devices (for example, Xiaomi, Lenovo, Oppo, LeEco, and so on) where the notifications sent via Firebase Cloud Messaging (FCM)/Google Cloud Messaging (GCM) remain undelivered. For more information, refer to [Delivery Optimization](https://docs.clevertap.com/docs/delivery-optimization).

###Roles
A specific system role dedicated to each user. CleverTap offers role-based access control (RBAC) for account administrators to enable different levels of access for dashboard users. For more information, refer to [User Roles](https://docs.clevertap.com/docs/role-based-access-control#user-roles).

##S
###Schema
An [organizational structure](https://docs.clevertap.com/docs/schema) of events, event properties, user properties, their values, and data sources.

###Segments
A group of users whose actions performed, actions not performed, or user profile properties match a set of defined criteria, over a period of time. For more information, refer to [Segments](https://docs.clevertap.com/docs/segments).

###Single Sign-On (SSO)
An authentication method that lets users access multiple applications and services with the use of a single set of login credentials. For more information, refer to [Single Sign-On (SSO)](https://docs.clevertap.com/docs/single-sign-on-sso).

##U
###User
A [user](https://docs.clevertap.com/docs/user-profiles) is a person who can access an application through a mobile app, web app, or web browser. A user identified in the CleverTap dashboard is a profile comprised of identifiers, properties, and events.

###User Identity
A [set of attributes](https://docs.clevertap.com/docs/user-profiles#user-profile-data-model) that uniquely identify a user (e.g., email, mobile number, custom identifier).

###User Property
An attribute that provides additional information about a user.
